Docker

1. Definition

Docker is a platform that enables containerization â€” packaging an application along with its dependencies, libraries, and configuration into a lightweight, portable container that can run anywhere.

2. Key Concepts

a) Container

  • A lightweight, standalone, and executable package of software.
  • Includes everything needed to run the application: code, runtime, system tools, libraries, and settings.
  • Runs consistently across different environments.

b) Image

  • A read-only template used to create containers.
  • Can contain an OS, application code, libraries, and dependencies.
  • Docker images are stored in registries like Docker Hub.

c) Docker Engine

  • The runtime that runs and manages Docker containers.
  • Consists of:
    • Docker Daemon: Background service that manages containers.
    • Docker CLI: Command-line tool to interact with Docker.
    • REST API: Interface for automation and integration.

d) Dockerfile

  • A text file containing instructions to build a Docker image.
  • Defines:
    • Base image
    • Application code
    • Dependencies
    • Commands to run

e) Docker Compose

  • Tool to define and manage multi-container applications.
  • Uses a YAML file to configure services, networks, and volumes.

Docker vs virtual machines

FeatureDocker (Container)VM (Virtual Machine)
OverheadLightweight, shares OS kernelHeavy, full OS per VM
Boot TimeSecondsMinutes
PortabilityHigh, runs anywhereLower, dependent on hypervisor
IsolationProcess-levelHardware-level
Use CaseMicroservices, DevOps, CI/CDLegacy apps, full OS environments
